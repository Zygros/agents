# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it by emailing the maintainer directly or by opening a private security advisory on GitHub.

**Please do not open a public issue for security vulnerabilities.**

### What to Include

When reporting a vulnerability, please include:

- A description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Any suggested fixes (if applicable)

### Response Time

We aim to respond to security reports within 48 hours and will work to address critical vulnerabilities as quickly as possible.

## Supported Versions

We provide security updates for the latest release. Older versions may not receive security patches.

## Security Best Practices

When using this project:

- Never commit API keys, passwords, or other sensitive credentials
- Use environment variables for configuration
- Keep dependencies up to date
- Follow the principle of least privilege
